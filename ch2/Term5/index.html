    📌 条款05：揭秘C++自动生成的成员函数

    🚨 编译器自动补全的四大函数

    空类 ≠ 真正空类！编译器会为未声明以下函数的类自动生成：

    class Empty {
public:
    Empty();                    // 默认构造函数
    Empty(const Empty&);        // 拷贝构造函数
    ~Empty();                   // 析构函数（非虚）
    Empty& operator=(const Empty&); // 拷贝赋值运算符
};

    🎯 自动生成函数的行为解析

    默认构造函数

    • 调用基类和非静态成员的默认构造函数
    • 不初始化内置类型（int/double等）

    拷贝构造函数

    • 按成员逐位拷贝（bitwise copy）
    • 调用成员对象的拷贝构造函数
    class Example {
    std::string str; 
    int num;
};
// 编译器生成拷贝构造函数等价于：
Example(const Example& rhs) : str(rhs.str), num(rhs.num) {}

    拷贝赋值运算符

    // 编译器生成版本等价于：
Example& operator=(const Example& rhs) {
    str = rhs.str;
    num = rhs.num;
    return *this;
}

    析构函数

    • 调用基类和非静态成员的析构函数
    • 非虚析构（除非基类有虚析构）

    🚩 特殊场景处理规则

    ❌ 拒绝生成拷贝赋值运算符的情况

    graph TD
    A[尝试生成拷贝赋值运算符] --> B{是否合法?}
    B -->|含引用成员| C[拒绝生成]
    B -->|含const成员| C
    B -->|基类赋值运算符private| C
    B -->|其他合法情况| D[正常生成]

    示例：引用成员引发的问题

    class RefMember {
public:
    RefMember(std::string& name) : nameRef(name) {}
private:
    std::string& nameRef;  // 引用成员
    const int id;          // const成员
};

RefMember a(name1);
RefMember b(name2);
a = b;  // 编译错误！无法生成拷贝赋值运算符

    💡 关键要点总结

    📢 编译器生成函数四定律：
1️⃣ 空类自动补全基础函数
2️⃣ 内含引用/const需自定赋值
3️⃣ 派生类受制基类权限
4️⃣ 特殊成员需手动掌控

    必须手动定义的情况

    场景 需定义的函数
    类含引用成员 拷贝赋值运算符
    类含const成员 拷贝赋值运算符
    基类赋值运算符private 派生类拷贝赋值运算符
    需要资源管理 所有特殊成员函数

    最佳实践建议

    1. 显式声明需要使用的特殊成员函数
    2. 使用= default明确要求编译器生成默认版本
    class SafeClass {
public:
    SafeClass() = default;
    SafeClass(const SafeClass&) = default;
    SafeClass& operator=(const SafeClass&) = default;
    ~SafeClass() = default;
};

    🔚 理解编译器行为是写出稳健类的第一步！当遇到特殊成员时，切记手动接管控制权。
