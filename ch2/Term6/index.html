    📌 条款06：明确禁用编译器自动生成的函数

    🚨 问题场景：禁止对象拷贝

    某些类（如房地产系统中的HomeForSale）需要禁止拷贝行为：

    HomeForSale h1;
HomeForSale h2(h1);  // ❌ 应禁止拷贝构造
HomeForSale h3 = h1; // ❌ 应禁止拷贝赋值

    🛡️ 解决方案一：私有化拷贝函数

    实现步骤

    class HomeForSale {
public:
    // ...其他成员函数
private:
    HomeForSale(const HomeForSale&);            // 仅声明不实现
    HomeForSale& operator=(const HomeForSale&); // 仅声明不实现
};

    防御效果对比

    调用场景 错误阶段 错误类型
    外部代码尝试拷贝 编译期 访问权限错误
    成员/友元函数尝试拷贝 链接期 未定义符号错误

    🛠️ 解决方案二：使用不可拷贝基类

    Uncopyable基类实现

    class Uncopyable {
protected:
    Uncopyable() = default;
    ~Uncopyable() = default;
private:
    Uncopyable(const Uncopyable&);            // 禁用拷贝构造
    Uncopyable& operator=(const Uncopyable&); // 禁用拷贝赋值
};

    继承应用方式

    class HomeForSale : private Uncopyable {  // 私有继承
public:
    // ...无需声明拷贝函数
};

    Boost库增强版

    #include <boost/noncopyable.hpp>
class HomeForSale : boost::noncopyable { /*...*/ };

    💡 关键要点总结

    📢 禁用拷贝双刃剑：
1️⃣ 私有声明断生成
2️⃣ 基类继承更干净
3️⃣ Boost实现更优雅
4️⃣ C++11可用=delete

    方法对比表

    方法 优点 缺点
    私有化声明 简单直接 需要手动添加声明
    Uncopyable基类 代码复用 可能影响类继承体系
    boost::noncopyable 标准化解决方案 需要外部依赖
    C++11 =delete 最直观的现代语法 需要C++11及以上支持

    现代C++推荐做法（C++11+）

    class HomeForSale {
public:
    HomeForSale(const HomeForSale&) = delete;
    HomeForSale& operator=(const HomeForSale&) = delete;
};

    1. 继承方式选择：
    2. 优先使用private继承（见条款32）

    4. 空基类优化： mermaid graph LR A[Uncopyable] --> B[HomeForSale] C[其他基类] --> B

    5. 可能影响空基类优化（见条款39）

    6. 多重继承时需注意布局（见条款40）

    8. 使用基类方案可将错误从链接期提前到编译期
