    📌 条款04：确保使用的是已经初始化的对象

    🚨 未初始化对象的危害

    1. 读取未初始化值会导致未定义行为，可能引发程序崩溃或产生不可预测结果
    2. C++初始化规则复杂性：
    3. C风格部分（如数组）不保证初始化
    4. 非C风格部分（如vector）保证初始化

    🎯 初始化最佳实践

    内置类型处理

    👉 必须手工初始化：

    int x = 0;                        // 整型初始化
const char* text = "C-style str"; // 指针初始化
double d;
std::cin >> d;                    // 运行时初始化

    类成员初始化

    ❌ 错误示范（赋值非初始化）

    class ABEntry {
public:
    ABEntry(const std::string& name, const std::string& address) {
        theName = name;    // 赋值操作
        theAddress = address;
    }
private:
    std::string theName;
    std::string theAddress;
};

    ✅ 正确做法（成员初始化列表）

    class ABEntry {
public:
    ABEntry(const std::string& name, const std::string& address)
        : theName(name),     // 初始化操作
          theAddress(address) 
    {} 
private:
    std::string theName;
    std::string theAddress;
};

    🌐 初始化顺序问题

    跨编译单元难题

    graph TD
    A[编译单元A] -->|使用| B[编译单元B的non-local static对象]
    B -->|可能未初始化| C[运行时错误]

    ✅ 解决方案（Singleton模式）

    class FileSystem {
public:
    static FileSystem& instance() {
        static FileSystem fs;  // 首次调用时初始化
        return fs;
    }
};

// 使用方式
FileSystem::instance().operation();

    🚩 核心原则口诀

    📢 初始化三定律：
1️⃣ 基础类型要手管
2️⃣ 成员列表效率赞
3️⃣ 静态对象本地换

    💡 关键要点

    1. 内置类型必须显式初始化
    2. 优先使用成员初始化列表
    3. 声明顺序决定初始化顺序
    4. 用local static替代non-local static
    5. 多线程环境需特别注意静态对象初始化

    🔚 遵循这些准则，可有效避免90%以上的对象初始化相关问题！
